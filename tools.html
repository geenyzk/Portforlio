Tool Stack

A lean toolchain that ships polish.

From idea to launch, I keep my stack intentionally small: design in Figma or Framer, build fast in React, ship backends with Django, and style with the right layer of CSS—vanilla, Tailwind, or modules—depending on the project’s needs.

Launch checklist

Build flow
  1. 01 Wireframes & clickable prototype in Figma/Framer.
  2. 02 Design tokens + components, CSS approach chosen per scope.
  3. 03 React build with accessible, reusable blocks.
  4. 04 API layer on Django; Node.js scripts for tooling.

How I choose the layer

Decision flow

Rapid concept: Figma or Framer → Tailwind → React.

Structured product: Figma components → CSS Modules for scoping → React.

Data-heavy: Django API → React views → modular CSS for performance.

Motion-led page: Framer prototype → Tailwind + CSS custom props → React + small GSAP/Framer Motion.

Each project gets the smallest possible set of dependencies so shipping stays quick and maintenance stays simple.
